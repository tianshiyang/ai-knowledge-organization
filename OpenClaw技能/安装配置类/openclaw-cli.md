## 一句话定位

跨平台操作和故障排除 OpenClaw CLI 的综合技能，适合把安装、配置、消息收发、插件和系统健康检查串成一套排障流程。

## 适合谁

- 正在部署或维护 OpenClaw 的开发者
- 需要经常操作 Gateway、Node、Channel 的使用者
- 想把 OpenClaw CLI 作为主要入口的人

## 典型使用场景

- OpenClaw CLI 日常使用
- Gateway 和 Node 生命周期管理
- Channel 登录和配置
- 消息收发调试
- Agent 轮次管理
- 模型配置和切换
- 插件管理
- 系统健康检查

## 为什么值得用

- 覆盖 OpenClaw CLI 的关键操作链路，适合做统一入口。
- 比单点命令说明更适合排障，尤其是配置、消息和 Agent 轮次混在一起出问题时。
- 对平台专项内容来说，这类技能比泛泛的“安装文档”更实用。

## 前端视角说明

对前端开发本身帮助有限，但如果你自己也负责 OpenClaw 环境、Node 节点或插件接入，这会比纯平台文档更省时间。

## 通用性说明

这不是通用工作流技能，主要服务 OpenClaw 本体维护和 CLI 操作。

## 风险与注意事项

- 适合平台专项场景，不建议和“前端常用技能”混为一类。
- 覆盖面较广，使用时要明确自己是在处理安装、消息、模型还是插件问题。

## 安装方式

```bash
clawhub install openclaw-cli
```
