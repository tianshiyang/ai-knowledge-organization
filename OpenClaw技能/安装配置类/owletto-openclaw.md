## 一句话定位

为 OpenClaw 接入 Owletto 记忆能力的插件配置技能，适合需要长期上下文记忆的用户。

## 适合谁

- 希望给 OpenClaw 增加长期记忆能力的人
- 需要配置 Owletto 插件和 OAuth 登录的用户
- 同时在意 MCP 健康状态和插件联通性的使用者

## 典型使用场景

- 需要为 OpenClaw 添加记忆功能
- Owletto 插件安装
- OAuth 认证配置
- MCP 健康检查

## 为什么值得用

- 记忆能力是很多 Agent 工作流从“一次性对话”走向“长期协作”的关键一步。
- 这个技能把插件、登录和健康验证放在同一个流程里，实用性比散装步骤高。

## 前端视角说明

如果你用 OpenClaw 做长期项目协作、知识整理或代码上下文延续，这个技能会很有帮助；否则优先级一般。

## 通用性说明

它属于插件集成类技能，不是通用高频技能，但对长期协作工作流很有价值。

## 风险与注意事项

- 涉及 OAuth 登录和外部插件接入。
- 引入记忆后，要额外关注数据边界和持久化内容。

## 安装方式

```bash
clawhub install owletto-openclaw
```
