# 进阶与谨慎使用

这页专门收录那些“能力很强，但不建议直接作为默认推荐”的 OpenClaw 技能。它们通常有以下特征：

- 权限范围更大
- 能操作系统、网络、浏览器或外部服务
- 会批量安装第三方技能
- 需要你先看清说明，再决定是否真的要装

它们不是不能用，而是不适合直接放进“新手必装”。

## 技能清单

| 技能 | 为什么强 | 为什么要谨慎 | 建议用法 |
|------|------|------|------|
| [Browser Automation CLI](https://clawhub.ai/skills/browser-pc) | 可自动导航网页、截图、抓取内容、执行表单操作 | ClawHub 页面提示其运行要求和配置方式需要额外审查，权限面较大 | 只在联调、自动化回归、网页操作明确需要时启用 |
| [openclaw-plus](https://clawhub.ai/skills/openclaw-plus) | 集成 Python 执行、包管理、git、抓网页、调 API 等能力 | 属于系统级通用工具箱，能力太全，误用成本也更高 | 适合作为高阶个人工具箱，不建议团队默认全装 |
| [OpenClaw Master Skills](https://clawhub.ai/skills/openclaw-master-skills) | 可作为技能索引和批量发现入口 | 会拉取大量第三方技能，安全和质量要逐个审查 | 先用它发现，再手动精选安装 |
| [Agency Agents Openclaw](https://clawhub.ai/skills/agency-agents-openclaw) | 多 Agent 编排完整，能覆盖工程、设计、营销、项目管理 | 角色很多，容易让输出看起来完整但失去任务聚焦 | 适合复杂项目的编排，不适合简单任务默认开启 |
| [Visual Explainer for OpenClaw](https://clawhub.ai/skills/visual-explainer-openclaw) | 能把复杂方案转成漂亮的 HTML 页面 | 如果开启分享或外发流程，需要留意内容脱敏与发布范围 | 用于架构解释、复盘、提案时很值，但需人工审稿 |

## 使用原则

1. 先确认是否真的有这个能力需求，而不是因为“它看起来很强”就安装。
2. 先读 ClawHub 页面里的安全扫描和权限说明，再决定是否纳入常用工具链。
3. 涉及浏览器自动化、联网调用、批量安装、外发分享时，默认按“需要审查”处理。
4. 更适合先个人试用，再决定是否推荐给团队。

## 与核心推荐的区别

- 核心推荐强调“高频、低到中风险、长期复用”。
- 这页强调“能力强，但更需要边界感和人工判断”。
- 如果你刚开始整理自己的 OpenClaw 技能库，建议先看 [前端开发常用技能](../前端开发常用技能/index.md) 和 [通用常用技能](../通用常用技能/index.md)，再回来看这页。
