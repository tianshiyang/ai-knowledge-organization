## 一句话定位

对 OpenClaw Gateway 做策略和配置安全检查的审计型技能，适合在发布前或定期巡检时使用。

## 适合谁

- 负责 OpenClaw 环境安全的维护者
- 想定期做安全评估和配置巡检的人
- 在意合规与风险报告的使用者

## 典型使用场景

- 安全审计需求
- 合规性检查
- 发现潜在安全风险
- 定期安全评估

## 为什么值得用

- 很多 OpenClaw 问题不是“不能跑”，而是“能跑但不够安全”，这类技能能补足这个盲区。
- 适合做上线前检查或周期性自查。

## 前端视角说明

对纯前端开发帮助有限，但如果你的 OpenClaw 运行环境也由你负责，就值得纳入专项工具链。

## 通用性说明

不属于通用高频技能，偏平台专项安全审计。

## 风险与注意事项

- 它发现的是配置和策略风险，不代表能替代完整的安全审计流程。
- 如果环境涉及外网暴露、节点联通和插件权限，应结合运维技能一起看。

## 安装方式

```bash
clawhub install openclaw-security-policy-check
```
