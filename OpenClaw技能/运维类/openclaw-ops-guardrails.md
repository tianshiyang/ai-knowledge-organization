## 一句话定位

面向 OpenClaw 运维防呆和标准化巡检的技能，适合跨设备环境、远程执行不稳定和发布前检查。

## 适合谁

- 维护多节点或跨设备 OpenClaw 环境的人
- 经常遇到审批、配对、远程执行异常的用户
- 想把排障和发布前检查流程标准化的人

## 典型使用场景

- 遇到 "又报错了 / 审批超时 / pairing required / system.run failed" 等运维问题
- 需要标准化运维规则和巡检流程
- 跨设备运维和远程执行场景

## 为什么值得用

- 它不只是排错，还试图把“防呆、巡检、脱敏检查”做成一套标准动作。
- 对长期维护 OpenClaw 的用户，比一次性修复技能更有体系感。

## 前端视角说明

如果你只是把 OpenClaw 当工具，这个技能大概率不是高频；如果你自己维护 Gateway 和节点环境，它会很值。

## 通用性说明

高度垂直于 OpenClaw 运维，不属于通用技能。

## 风险与注意事项

- 适合较复杂的运维环境，不适合拿来替代基础入门说明。
- 如果你没有多节点或远程执行场景，可以先不装。

## 安装方式

```bash
clawhub install openclaw-ops-guardrails
```
