# 关键工程点

这页专门对应面试题里的工程追问。

## 1. Memory 怎么设计

建议先区分：

- `短期状态`
  - `messages`
  - `tool_results`
  - `plan`
- `长期记忆`
  - 摘要
  - 实体
  - 向量检索

重点不是“记忆越多越好”，而是分层清楚。

## 2. 工具选择机制怎么做

可以从三类讲：

- LLM 根据工具描述自然语言选择
- 函数调用 / 结构化调用
- Router 显式分流

工具稳定性的关键不是 prompt，而是：

- 命名
- 描述
- 参数
- 返回结构

## 3. 安全与权限

真实工程里要强调：

- 细粒度权限
- 读写分层
- 高风险动作确认
- 工具输入输出校验
- 审计日志

## 4. 评估指标

至少分成：

- 任务完成率
- 工具调用正确率
- 路由或计划正确率
- 成本和时延
- 人工接管率

## 5. 死循环防护

至少同时有：

- 最大轮数
- 最大工具调用数
- 超时
- 连续无进展检测

## 面试里的推荐说法

“复杂 Agent 的稳定性来自工具设计、状态设计、控制流、权限边界和评估，而不只是 prompt 本身。”
