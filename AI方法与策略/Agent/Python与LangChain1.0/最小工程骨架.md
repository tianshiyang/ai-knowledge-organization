# 最小工程骨架

这份骨架不是生产级模板，而是面试里“能讲清、能手写、能继续扩展”的结构。

## 推荐目录

```text
agent_app/
├── app.py
├── prompts.py
├── agents/
│   ├── simple_agent.py
│   └── supervisor_agent.py
├── tools/
│   ├── search_tools.py
│   └── business_tools.py
├── graphs/
│   ├── router_graph.py
│   ├── planner_graph.py
│   └── reflection_graph.py
├── memory/
│   ├── short_term.py
│   └── long_term.py
└── evals/
    ├── smoke_eval.py
    └── regression_eval.py
```

## 每层职责

### `agents/`

- 放 `create_agent` 组装逻辑
- 区分普通 Agent 和 supervisor Agent

### `tools/`

- 放工具定义
- 控制工具边界和返回结构

### `graphs/`

- 放 `LangGraph` 编排
- 适合 Router、Plan-and-Execute、Reflection

### `memory/`

- 区分短期状态和长期记忆
- 避免把两者混在一起

### `evals/`

- 放回归测试和离线评测脚本

## 面试回答话术

“我会把 Agent 组装、工具、图编排、记忆和评测拆开。这样一方面主链路清晰，另一方面后续加路由、人审和回归测试都更自然。”
